# Slickplan — Content Workflow: 9 Steps

## Summary

Focuses on building the workflow infrastructure before production begins. Unique in that it includes workflow design (diagramming, user flow mapping, role assignment) as explicit tasks, not assumed prerequisites.

## 9 Steps

1. Identify goals, requirements, and constraints
2. Define tasks in the production process (every step catalogued)
3. Assign a workflow leader (managing editor — owns briefs, approvals, conflict resolution)
4. Identify the content team (skills audit, gap identification)
5. Create a strategy workflow diagram (bird's-eye visualization)
6. Define start and end points (where content originates, when it's truly done)
7. Diagram user flow (how visitors navigate — reveals content gaps, dead ends)
8. Order and delegate tasks (assign responsibilities, minimize handoffs)
9. Build revisions into workflow (review cycles locked in, critical elements frozen before dependent steps)

## What It Contributes

Strongest source on the operational setup phase — what has to be done before the first piece of content is ever written.
