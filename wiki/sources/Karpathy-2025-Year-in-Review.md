# Karpathy 2025 LLM Year in Review

**Source**: [karpathy.bearblog.dev](https://karpathy.bearblog.dev/year-in-review-2025/) | Andrej Karpathy | December 2025

---

## Summary

Karpathy's annual reflection on LLM progress and implications for software development. Key focus: vibe coding as a democratization threshold, the pivot to agentic engineering as LLMs matured, and predictions for how developer roles will bifurcate.

---

## Key Claims

**On vibe coding democratization**:
"Programming is not strictly reserved for highly trained professionals, it is something anyone can do." Karpathy demonstrates vibe coding across real projects — a custom BPE tokenizer in Rust, multiple demos, ephemeral debugging tools — treating code as "free, ephemeral, malleable, discardable." (confidence: high)

**On software industry transformation**:
"Vibe coding will terraform software and alter job descriptions." Not a marginal shift — roles, workflows, and org structures around development will change materially. (confidence: high)

**On local vs. cloud agents**:
Running agents locally on the developer's machine outperforms cloud-deployed alternatives because local provides immediate access to "already-existing and booted up computer, its installation, context, data, secrets, configuration, and the low-latency interaction." This is the architectural justification for tools like Claude Code. (confidence: high)

**On the future of developer roles**:
A bifurcated future: LLM labs produce generally capable models; specialized applications "organize, finetune and actually animate teams of them into deployed professionals in specific verticals." Developers shift toward orchestration and domain specialization rather than foundational coding. (confidence: medium — stated as prediction)
