# Content Marketing Workflow

A content marketing workflow is the structured system that governs how content moves from idea to published asset to measured result. It defines roles, task sequences, handoffs, and quality gates.

## Two Workflow Types

**Task-based workflow** — tracks every discrete step in the production process (ideation → brief → draft → edit → design → approve → publish). Progress is measured by task completion.

**Status-based workflow** — tracks content by its current stage (Briefing / Drafting / In Review / Approved / Scheduled / Published). Simpler to manage at volume.

Most teams use a hybrid: status stages as the macro view, task lists inside each stage.

## Core Workflow Roles

| Role | Responsibility |
|---|---|
| Content Strategist | Sets goals, builds calendar, defines pillars, aligns to business priorities |
| Managing Editor / Workflow Leader | Owns briefs, approvals, conflict resolution, editorial standards |
| Writer / Creator | Produces drafts |
| Editor | Reviews for accuracy, clarity, brand voice |
| SEO Specialist | Keyword targeting, on-page optimization |
| Designer | Visual assets, thumbnails, infographics |
| Publisher / CMS Manager | Schedules, formats, and publishes final content |

## Non-Negotiables for Scale

- One designated workflow leader with authority to approve and unblock
- Written briefs before any content is created
- Revision cycles locked in before production starts (not bolted on at the end)
- Content inventory updated after every publish

## Key Principle

The workflow is set up *before* the first piece is created — not constructed reactively as problems arise. (Source: [[Slickplan-Content-Workflow-9-Steps]])
