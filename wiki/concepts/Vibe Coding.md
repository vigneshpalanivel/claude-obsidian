# Vibe Coding

A software development practice in which the developer describes a desired program in natural language and accepts AI-generated code without reviewing it thoroughly — relying on results and iterative follow-up prompts rather than manual code inspection.

Coined by [[Andrej Karpathy]] in February 2025. Named Collins English Dictionary Word of the Year for 2025. Merriam-Webster listed it in March 2025 as "slang & trending."

---

## Core Idea

"Where you fully give in to the vibes, embrace exponentials, and forget that the code even exists." — Karpathy

The developer guides, tests, and gives feedback on AI-generated code. The developer does not write it. Code is treated as "free, ephemeral, malleable, discardable."

---

## Key Characteristics

- Prompts drive implementation, not keystrokes
- Developer role: curator, tester, orchestrator — not author
- Suited to: prototyping, throwaway scripts, one-off tools, internal automation
- Risky for: production systems, security-critical code, long-lived codebases

---

## Adoption Data

- 25% of Y Combinator Winter 2025 startups had codebases that were 95%+ AI-generated
- AI coding tools reached 90% adoption in engineering teams by end of 2025 (up from 61% a year prior)

---

## Limitations

- AI code has 1.7x more bugs than manually written code
- AI code causes roughly 1 in 5 security breaches
- Not appropriate for production systems without review, testing, and human judgment
- "Forget the code even exists" is viable for throwaway projects; dangerous for long-lived products

---

## Evolution: Beyond Vibe Coding

By late 2025, Karpathy retired the term "vibe coding" in favor of [[Agentic Engineering]] — a more mature framing where developers orchestrate autonomous agent workflows rather than accepting raw LLM output verbatim.
